# MET Mythic 문항 설계 원칙
## Question Design Principles v1.0

---

## 1. 핵심 철학

### 1.1 동기 vs 성향

```
❌ 성향(Personality) 측정:
"당신은 계획적인 편인가요?"
"마감 전에 일을 끝내는 편인가요?"

✅ 동기(Motivation) 측정:
"마감 전날, 무엇이 당신을 움직이게 했나요?"
"일을 끝내고 나면 어떤 느낌이 드나요?"
```

**차이점:**
- 성향: "어떻게 행동하는가" (HOW)
- 동기: "왜 움직이는가" (WHY)

### 1.2 에너지 기반 측정

모든 문항은 **에너지 흐름**을 측정합니다:
- 언제 에너지가 **충전**되는가?
- 언제 에너지가 **고갈**되는가?
- 무엇이 엔진을 **점화**시키는가?

## 2. 문항 유형 4가지

### 2.1 선택형 (Choice) — 4지선다

**특징:**
- 각 선택지가 **다른 동기**에 매핑
- 가장 솔직한 첫 반응 선택
- 점수: 선택한 동기에 +1

**구조:**
```typescript
{
  type: 'choice',
  text: "상황 제시 질문",
  options: [
    { id: 'A', text: "반응 A", motive: 'achievement' },
    { id: 'B', text: "반응 B", motive: 'mastery' },
    { id: 'C', text: "반응 C", motive: 'creation' },
    { id: 'D', text: "반응 D", motive: 'connection' },
  ]
}
```

**예시:**
```
Q: 복잡한 문제를 만났을 때 가장 먼저 드는 느낌은?

A) 풀고 싶다, 재밌겠다 → Mastery
B) 빨리 해결하고 싶다 → Achievement  
C) 새로운 방법을 찾고 싶다 → Creation
D) 누가 같이 했으면 → Connection
```

---

### 2.2 정도형 (Likert) — 5점 척도

**특징:**
- 하나의 동기에 대한 **동의 정도** 측정
- 1(전혀 아니다) ~ 5(매우 그렇다)
- 접근형/회피형 버전 별도

**구조:**
```typescript
{
  type: 'likert',
  text: "동기 관련 진술문",
  motive: 'achievement',
  direction: 'approach', // 또는 'avoidance'
  options: [
    { value: 1, text: "전혀 그렇지 않다" },
    { value: 2, text: "그렇지 않다" },
    { value: 3, text: "보통이다" },
    { value: 4, text: "그렇다" },
    { value: 5, text: "매우 그렇다" },
  ]
}
```

**예시 (접근형):**
```
Q: 목표를 달성하면 에너지가 충전된다.
→ Achievement (Approach)

1 ─ 2 ─ 3 ─ 4 ─ 5
```

**예시 (회피형):**
```
Q: 실패할 것 같으면 시작하기 어렵다.
→ Achievement (Avoidance)

1 ─ 2 ─ 3 ─ 4 ─ 5
```

---

### 2.3 시나리오형 (Scenario) — 상황 기반

**특징:**
- 구체적인 상황 제시
- 상황 속에서의 **반응/느낌** 선택
- 더 현실적인 동기 포착

**구조:**
```typescript
{
  type: 'scenario',
  text: "구체적 상황 설명",
  subtext: "질문",
  context: 'normal' | 'pressure' | 'burnout',
  options: [
    { id: 'A', text: "반응 A", motive: '...', value: 1 },
    // ...
  ]
}
```

**예시:**
```
상황: 팀 프로젝트가 끝났습니다. 
      결과는 성공적이었고, 팀원들이 축하합니다.

Q: 이 순간, 가장 먼저 드는 생각은?

A) "해냈다! 목표 달성!" → Achievement
B) "이번에 뭘 배웠지?" → Mastery
C) "다음엔 뭘 해볼까?" → Adventure
D) "팀원들 덕분이야" → Connection
```

---

### 2.4 양극형 (Bipolar) — 이중 선택

**특징:**
- 두 동기 중 **하나 선택** (강제 선택)
- 동기 충돌 상황 측정
- 또는 접근/회피 방향 측정

**구조:**
```typescript
{
  type: 'bipolar',
  text: "상황 또는 질문",
  options: [
    { id: 'A', text: "선택 A", motive: 'freedom' },
    { id: 'B', text: "선택 B", motive: 'security' },
  ]
}
```

**예시 (동기 충돌):**
```
Q: 안정적인 직장 vs 자유로운 프리랜서
   둘 중 하나만 선택해야 한다면?

A) 안정적인 직장 → Security
B) 자유로운 프리랜서 → Freedom
```

**예시 (접근/회피):**
```
Q: 마감이 다가올 때 나를 움직이게 하는 것은?

A) "끝내면 뿌듯할 거야" → Approach
B) "못 끝내면 큰일이야" → Avoidance
```

## 3. 카테고리별 설계 지침

### 3.1 동기 원천 (Motive Source) — 80문항

**목표:** 8개 동기 원천의 **강도** 측정

**구성 (원천당 10문항):**
- 선택형 3문항 (다른 동기와 비교)
- 리커트 접근형 3문항
- 리커트 회피형 2문항
- 시나리오형 2문항

**설계 원칙:**
```
✅ 좋은 문항:
"[동기]가 충족되면 에너지가 난다"
"[동기]가 없으면 힘이 빠진다"
"[상황]에서 [동기] 관련 반응"

❌ 나쁜 문항:
"당신은 [동기]를 중요하게 생각하나요?" (직접 질문)
"[동기]는 좋은 것이다" (사회적 바람직성)
```

---

### 3.2 점화 조건 (Ignition) — 30문항

**목표:** 6개 점화 조건의 **효과** 측정

**구성 (조건당 5문항):**
- 리커트형 3문항
- 시나리오형 2문항

**설계 원칙:**
```
✅ 좋은 문항:
"[조건]이 있으면 더 집중된다"
"[조건] 없이는 시작하기 어렵다"
"[조건 상황]에서의 에너지 변화"

❌ 나쁜 문항:
"[조건]을 좋아한다" (선호 vs 점화는 다름)
```

**조건별 핵심:**
| 조건 | 측정 포인트 |
|------|-------------|
| Competition | 경쟁자가 있을 때 에너지 ↑? |
| Complexity | 복잡할수록 몰입? |
| Deadline | 마감이 있어야 움직임? |
| Audience | 지켜보는 사람이 있으면? |
| Autonomy | 자율권이 있어야? |
| Crisis | 위기 상황에서 각성? |

---

### 3.3 방향 (Direction) — 32문항

**목표:** 접근/회피 **패턴** 측정

**구성 (원천당 4문항):**
- 접근형 2문항
- 회피형 2문항

**설계 원칙:**
```
접근형 (Toward):
"[긍정적 결과]를 얻고 싶다"
"[긍정적 상태]가 되면 좋겠다"

회피형 (Away):
"[부정적 결과]를 피하고 싶다"  
"[부정적 상태]가 되면 안 된다"
```

**예시 (Achievement):**
```
접근: "이기면 기분 좋다"
회피: "지면 안 된다"
```

---

### 3.4 동기 운영 (Operation) — 24문항

**목표:** 4개 축의 **성향** 측정

**구성 (축당 6문항):**
- 양극형 4문항
- 리커트형 2문항

**축별 양극:**
| 축 | 좌측 (-) | 우측 (+) |
|----|----------|----------|
| Rhythm | 계획형 | 즉흥형 |
| Recharge | 고독 충전 | 사회적 충전 |
| Release | 지구력형 | 폭발형 |
| Recovery | 빠른 회복 | 느린 회복 |

---

### 3.5 숨겨진 동기 (Shadow) — 24문항

**목표:** 표면 동기 vs 실제 동기 **불일치** 탐지

**설계 원칙:**
```
함정 문항 패턴:
1. 표면적으로 동기 A 측정처럼 보이지만
2. 실제로는 동기 B의 회피형을 측정

예시:
Q: "완벽하게 이해했는데 아무도 안 알아주면?"
   
표면: Mastery 측정처럼 보임
실제: Recognition (회피) 측정
   
A) 상관없다, 내가 알면 됐지 → 진짜 Mastery
B) 좀 서운하다 → Shadow Recognition
C) 알아주는 사람 찾아야지 → Recognition
D) 다음엔 더 잘 설명해야지 → Connection
```

---

### 3.6 성숙도 (Maturity) — 12문항

**목표:** Lv1~Lv4 **레벨** 판별

**설계 원칙:**
```
같은 동기라도 성숙도에 따라 다른 반응:

Achievement Lv1: "지면 끝이야"
Achievement Lv2: "이기고 싶어, 방법은 가려서"
Achievement Lv3: "지는 것도 배움"
Achievement Lv4: "이김/짐을 넘어 성장이 목표"
```

## 4. 응답 시간 활용

### 4.1 측정 방식
- 문항 표시 시점 → 선택 시점 (밀리초)
- 자동 저장

### 4.2 해석 기준
| 응답 시간 | 해석 |
|-----------|------|
| < 1초 | 의심 (무성의/패턴 응답) |
| 1-3초 | 직관적 반응 (신뢰도 높음) |
| 3-10초 | 숙고 (정상) |
| > 10초 | 갈등/어려움 (추가 분석) |

### 4.3 검증 활용
```typescript
// 너무 빠른 응답 비율
const suspiciousRate = answers.filter(a => a.responseTimeMs < 1000).length / answers.length;

// 30% 이상이면 경고
if (suspiciousRate > 0.3) {
  result.warning = '응답이 너무 빠릅니다. 신중하게 답변해주세요.';
}
```

## 5. Lite vs Full 배분 원칙

### Lite (100문항) — 15분
```
핵심만 측정:
- 동기 원천: 40문항 (원천당 5개)
- 점화 조건: 15문항
- 방향: 16문항
- 운영: 12문항
- 성숙도: 4문항
- 그림자: 5문항 (최소)
- 에너지: 8문항
```

### Full (300문항) — 40분
```
전체 측정:
- Lite 전부 포함
- 충돌 20문항
- 상황 30문항
- 오염 12문항
- 원형 층위 8문항
- 검증 8문항
- 추가 깊이 문항
```

## 6. 문항 작성 체크리스트

### 작성 전
- [ ] 측정하려는 동기/조건이 명확한가?
- [ ] 성향이 아닌 동기를 측정하는가?
- [ ] 에너지 흐름과 연결되는가?

### 작성 중
- [ ] 문장이 간결한가? (20자 내외)
- [ ] 선택지가 명확히 구분되는가?
- [ ] 사회적 바람직성 편향이 없는가?
- [ ] 중립적 표현인가?

### 작성 후
- [ ] 다른 문항과 중복되지 않는가?
- [ ] Lite/Full 배분이 적절한가?
- [ ] 함정 문항이면 표시했는가?

## 7. 금지 사항

```
❌ 절대 하지 말 것:

1. 직접 질문
   "성취가 중요한가요?" 
   
2. 이중 질문
   "목표를 세우고 달성하면 기분이 좋은가요?"
   
3. 유도 질문
   "대부분의 사람들은 인정받으면 기분이 좋습니다. 당신은?"
   
4. 극단적 표현
   "항상", "절대", "반드시"
   
5. 전문 용어
   "동기 원천", "점화 조건" 등 직접 노출
   
6. 부정형 + 부정 응답
   "인정받지 못하면 기분이 나쁘지 않다" (혼란)
```
